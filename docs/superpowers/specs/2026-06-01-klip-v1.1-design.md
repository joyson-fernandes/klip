# Klip v1.1 — Design Spec

Date: 2026-06-01

## Overview

v1.1 rebrands **gifsnap → klip** and adds three new capability surfaces plus a professional visual overhaul:

1. **Screenshot capture** — a second capture mode that produces a single PNG (no animation, no encoding).
2. **Annotation editor** — a markup window that opens automatically after every screenshot, with 11 tools in the primary toolbar.
3. **Configurable hotkeys** — both capture hotkeys are user-editable via the settings popover.
4. **Polish pass** — the existing GIF capture overlay and HUD get redesigned alongside the new editor so the whole app feels consistent with category leaders (CleanShot X, Shottr, Xnapper).

The existing GIF capture functionality is unchanged in behaviour but its appearance changes.

## Design language

Researched against CleanShot X, Shottr, Xnapper, ScreenSnap Pro, and Apple's ⌘⇧5 + Markup. Choices:

| Element | Decision |
|---|---|
| Materials | `NSVisualEffectView.material = .hudWindow` on floating toolbars and HUDs |
| Typography | System (`SF Pro`), 11–13 pt |
| Icons | SF Symbols only (no custom raster icons) |
| Accent | `#5E5CE6` (the project's existing indigo) |
| Corner radius | 8 pt on toolbars, 12 pt on the canvas image, 10 pt on the Quick Access thumbnail |
| Shadows | Two-stop: 1 pt ambient at 12% + 24 pt diffuse at 25% |
| Dim percentage | 45% black on region selector backdrop |
| Selection border | 1 px white inside, 1 px black outside (visible on any background, no marching ants) |
| Active tool indicator | Filled accent squircle behind the symbol; inactive tools at `secondaryLabelColor` |

## Feature 1 — Screenshot capture

### Hotkey

Default **⌘⇧2**. Pressing it triggers the existing `RegionSelector` flow. On `regionSelector(_:didSelect:on:)`:

- If the active hotkey was the GIF one → existing behaviour (start recording).
- If the active hotkey was the screenshot one → new code path: capture a single image of the region using `SCStream` with `queueDepth: 1` and one frame, OR via `CGWindowListCreateImage` for instant single-frame capture (no permission re-prompt overhead).

We use **`CGWindowListCreateImage` for screenshots**. It is synchronous, returns a `CGImage` directly, doesn't need a stream lifecycle, and works under the same Screen Recording permission grant. It also bypasses any frame-rate latency.

### Output

The captured `CGImage` is wrapped in `NSImage`, opened in the annotation editor. The user-facing flow is: hotkey → drag → press ↵ or release mouse → editor window opens with the screenshot loaded.

There is no "save and skip editor" option in v1.1. Every screenshot opens the editor — if the user just wants the raw screenshot, they hit ⌘S immediately.

## Feature 2 — Annotation editor

A separate `NSWindow` (not a panel, not full-screen) titled with the capture dimensions.

### Layout

A single `NSWindow` with three regions:

1. **Title bar** — system-styled `.unified` title. App name + dimensions on the left, Cancel + Save on the right. No custom title bar chrome.
2. **Canvas** — neutral `#1C1C1E` workspace, the screenshot centered with the two-stop drop shadow and a 12 pt corner radius. The canvas always grows to fill the window; scrolls when zoomed in.
3. **Floating toolbar pill** — *not* fixed to the window edge. A horizontal HUD-material pill anchored to the bottom-center of the canvas image (Shottr / CleanShot style). It floats over the canvas so it never reflows when the window resizes. Width slider + color swatches expand as a second pill below when a relevant tool is active.

### Tools (11 in primary toolbar)

Refined down from 14 based on research — toolbars over 11 icons feel cluttered. Lower-frequency tools (Speech bubble, Pixelate, Spotlight) move into a `···` overflow menu so the primary row never wraps.

**Primary row (left to right):**

| Group | Tools |
|---|---|
| Selection | Select |
| Shapes | Arrow, Rectangle, Ellipse, Line |
| Content | Pen (freehand), Text |
| Emphasis | Highlight, Blur |
| Numbering | Step counter (auto-incrementing 1, 2, 3…) |
| Canvas | Crop |

**Overflow menu (`···`):**
- Speech bubble
- Pixelate
- Spotlight

### Tool style

- **Colour palette**: 8 curated swatches (red, orange, yellow, green, blue, purple, black, white) + custom + eyedropper. The last 6 custom colours per tool persist via `UserDefaults`. Does NOT open the system `NSColorPanel`.
- **Width slider**: 1–20 px. Applies to stroke-based tools (arrow, line, rectangle, ellipse, pen).
- **Text** uses size + colour, no width.
- **Effects** (blur, pixelate, spotlight) have a single radius/intensity slider replacing the width slider when active.
- **Active tool**: filled `#5E5CE6` squircle behind the SF Symbol; inactive tools are `secondaryLabelColor`. Hover shows 6% white fill.

### Polish details

- **Snap guides** between annotation objects: an 8 pt grid plus center/edge alignment lines that appear during drag (Figma-style). None of the researched apps do this well — easy differentiator.
- **Selection handles** appear only on hover/pause for the selected shape.
- **Custom toast** for "Copied" / "Saved" — small accent-tinted pill anchored to the bottom-right of the editor window, auto-dismiss after 2 s, drag-to-dismiss earlier.

### Interaction

- Click a tool → cursor becomes the relevant icon.
- Click-drag on canvas → creates a new shape committed on mouse-up.
- Select tool → click an existing shape to select; arrow keys nudge; delete key removes; drag handles resize.
- ⌘Z / ⌘⇧Z → undo/redo.
- ⌘S → save (annotated PNG written to save folder + copied to clipboard + window closes + notification).
- Esc → cancel (window closes, nothing saved).

### Persistence

Annotations are not saved as editable layers — once the user hits Save, the final flattened PNG is written. There is no "open existing annotation file to keep editing" flow.

### Architecture

New components:

- `Screenshot` — captures a `CGImage` from a `CGRect` on a given `NSScreen`.
- `EditorWindowController` — owns the editor window's lifecycle.
- `AnnotationCanvas` — `NSView` subclass that renders the screenshot bitmap + a list of `Annotation` model objects.
- `Annotation` — protocol with concrete types per drawn tool: `ArrowAnnotation`, `RectAnnotation`, `EllipseAnnotation`, `LineAnnotation`, `PenAnnotation`, `TextAnnotation`, `SpeechAnnotation`, `StepAnnotation`, `HighlightAnnotation`, `BlurAnnotation`, `PixelateAnnotation`, `SpotlightAnnotation`.
- **Crop** is not an `Annotation`. It mutates the canvas extent itself: the user drags a crop rectangle, presses ↵, and the screenshot bitmap + every existing annotation's coordinates are translated to the new origin. Crop history pushes a single undoable `CropOperation` onto the undo stack alongside annotations.
- `ToolbarView` — SwiftUI view with tool grid + colour swatches + width slider.
- `EditorState` — `ObservableObject` holding selected tool, colour, width, annotation list, undo/redo stacks.
- `AnnotationRenderer` — flattens canvas to PNG `Data` using `NSImage(size:).draw(in:)` + `bitmapImageRepForCachingDisplay`.

## Polish — region selector overlay

Existing overlay gets a redesign for consistency with the rest of the app:

- **Backdrop dim** drops from 40% to 45% black (research-validated middle ground).
- **Selection border**: 1 px white inside + 1 px black outer shadow — visible on any background. No marching ants.
- **Dimensions HUD**: small rounded chip with `.hudWindow` material (not a solid purple block), anchored to the bottom-right of the selection. Flips to top-right when within 40 pt of the screen edge. Click toggles between logical and physical pixels.
- **Edge handles**: 8 pt squares at the four corners that appear on hover/pause for fine resize.
- **Keyboard**: Esc cancels, Return confirms, arrow keys nudge the selection by 1 px (Shift+arrow = 10 px), Space-drag moves the entire selection without resizing.
- **Magnifier loupe** (out of scope for v1.1 but architected for): 120 pt circular loupe at cursor with 8× zoom + RGB hex readout. Wired but disabled behind a settings toggle initially.

## Polish — recording HUD

Existing HUD pill stays, but restyled:

- Material: `.hudWindow` blur instead of solid `rgba(0.1, 0.1, 0.1, 0.9)`.
- Border: 1 pt inner stroke at 8% white.
- Red dot becomes a `circle.fill` SF Symbol with `.symbolEffect(.pulse)` (macOS 14+) or a manual core-animation pulse fallback.
- Stop instruction updates to use the user's configured hotkey label, not hard-coded "⌘⇧G".

## Polish — Quick Access thumbnail (new)

After every capture (screenshot or GIF), a small floating thumbnail appears in the bottom-right of the active display (160 × 100 pt, 10 pt radius, `.hudWindow` material). It:

- Auto-fades after **6 s** unless the mouse hovers over it
- Click → opens the file (annotation editor for screenshots, Preview/QuickLook for GIFs)
- Drag → standard `NSDraggingItem` carrying the file URL — drop into any app
- Right-click → menu with "Copy", "Show in Finder", "Delete"
- Dismissible with a small × on hover

This is the single most-praised interaction in this category (CleanShot / ScreenSnap) and worth adding even in v1.1.

## Feature 3 — Configurable hotkeys

### Popover changes

A new **Hotkeys** section added below the Save folder row:

```
HOTKEYS
Screenshot     [⌘ ⇧ 2]   ⓧ
Record GIF     [⌘ ⇧ G]   ⓧ
                 Reset to defaults
```

Each row shows the current combo in a recorder field. Click the field → it shows "Press combo…" and listens for the next key event. Press combo → field updates. Click ⓧ → clears the binding (no hotkey assigned). Reset link restores both defaults.

### Implementation

- A `HotkeyRecorder` SwiftUI view binds to a `KeyCombo?` (struct: `keyCode: UInt32`, `modifiers: UInt32`).
- `SettingsStore` adds two new properties: `screenshotHotkey: KeyCombo?`, `gifHotkey: KeyCombo?` — encoded as JSON in UserDefaults.
- `MenuBarController` is refactored: instead of one hardcoded hotkey, it owns two `RegisteredHotkey` references and a small `HotkeyManager` that registers, unregisters, and rebinds via Carbon when settings change.
- `HotkeyManager` listens to `SettingsStoreObservable` via Combine — any change reregisters the affected hotkey.

### Validation

- The recorder ignores combos with no modifier key (would conflict with normal typing).
- The recorder allows the same combo for both — the user is free to make it confusing.
- We do not detect system-shortcut conflicts. macOS will silently fail to register; we surface that as a notification once on app launch if either hotkey didn't register.

## Out of scope (still)

- Video / MP4 recording
- Cloud upload (Imgur, S3)
- App Store distribution
- Multi-monitor capture spanning displays
- Editing existing image files (only freshly captured screenshots)
- Annotation on GIFs (already decided in brainstorming)

## Migration plan

1. Rename source folders: `gifsnap/` → `klip/`, `gifsnapTests/` → `klipTests/`. Xcode scheme renamed accordingly. `project.yml` target renamed.
2. Bundle ID changes to `com.joyson.klip` — old TCC grants under the old bundle ID become orphaned. Users will be prompted once for Screen Recording on first v1.1 launch. Acceptable since the user base is the developer themselves.
3. State files in the existing medium-sync repo are unaffected (different project).
4. Old `gifsnap.app` removed from `/Applications/` during the install; new `klip.app` takes its place.
